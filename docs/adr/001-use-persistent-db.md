# ADR 001: Использование постоянной БД вместо JSON-файлов

## Статус

Принято.

## Контекст

Изначально приложение сохраняло:

- результаты анализа;
- ошибки;
- сценарии;
- настройки

в JSON-файлы внутри директории приложения.

Это приводило к рискам:

- потери данных при деплое;
- отсутствия устойчивого источника истины;
- неудобной поддержки очереди;
- сложностей с дальнейшим развитием.

## Решение

Перейти на постоянное хранилище в виде SQLite-совместимой БД через `sql.js`.

Файл БД хранить вне директории релиза, например:

```text
/var/lib/callsreport/callsreport.sqlite
```

При первом запуске нового слоя хранения:

- создать схему;
- импортировать данные из legacy JSON, если БД пуста;
- дальше использовать только БД.

## Последствия

### Положительные

- данные не теряются при пересборке приложения;
- единая структура хранения;
- можно хранить очередь jobs и ошибки;
- проще развивать систему дальше.

### Ограничения

- решение рассчитано на один активный экземпляр процесса;
- payload пока хранится частично денормализованно;
- для более сложного масштаба в будущем может потребоваться внешняя СУБД.

## Что нужно соблюдать

- не хранить БД внутри релизной директории;
- документировать будущие миграции;
- перед изменением схемы делать backup БД.
